# 自我进化

## 目标

从**本次会话**和**近期短期记忆**中提炼有价值的信息，沉淀到对应的长期文件中。

> 这是 Wopal 对抗遗忘诅咒的核心机制——不只是记录发生了什么，而是从经验中**学习**。

## 三类沉淀物

| 类型 | 本质 | 判断标准 | 示例 |
|------|------|---------|------|
| **知识** | 是什么（事实） | "这是**需要记住的事实**吗？" | "OpenCode 非交互模式权限会被 auto-reject" |
| **经验** | 怎么做更好 | "这是**有效的做法**吗？" | "复杂变更拆分为独立小变更，提升并行效率" |
| **教训** | 不要怎么做 | "这是**需要避免的坑**吗？" | "fetch 不设置超时会无限挂起" |

## 流程

### 1. 扫描信息源

同时扫描：
- **本次会话**：关键事件、决策、教训
- **短期记忆**：`memory/diary/` 下近 3 天的流水
- **用户偏好信号**（→ USER.md）：
  - 用户拒绝方案时的理由
  - 用户对做法的反馈（不满/认可）
  - 用户的沟通风格反馈
  - 用户明确说"记住"/"下次注意"
  - 用户反复纠正同一类问题

**跳过已标记条目**：`[进化吸收]` `[进化排除]`

### 2. 提取与分类

从信息源中识别可沉淀的信息，归类为知识/经验/教训。

**区分规则 vs 沉淀**：
- 规则（永久约束）→ 归 `AGENTS.md`
- 沉淀（知识/经验/教训）→ 归 `MEMORY.md`

### 3. 确定目标文件

| 目标文件 | 写入什么 |
|---------|---------|
| `USER.md` | 用户偏好、沟通方式 |
| `AGENTS.md` | 永久遵守的工作规则 |
| `MEMORY.md` | 隐性知识、操作经验、负向警示 |
| 子项目 `AGENTS.md` | 子项目内部的技术栈、结构 |

### 4. 展示进化方案

```
🧬 进化方案：

📝 MEMORY.md
  - [知识] OpenCode 非交互模式权限会被 auto-reject
  - [经验] 复杂变更拆分为独立小变更，提升并行效率
  - [教训] fetch 必须设置超时，否则会无限挂起

📐 .workspace.md
  - [知识] 新增 tools/process/ 目录

确认执行？
```

### 5. 执行写入与标记

用户确认后：
1. 写入对应长期文件
2. 在源文件中标记：
   - 已进化 → `[进化吸收]`
   - 无需进化 → `[进化排除]`

### 6. 日记归档

将 3 天前的日记移动到 `memory/diary/archived/`：
```bash
# 获取 3 天前的日期
cutoff_date=$(date -v-3d '+%Y-%m-%d')

# 移动旧日记到归档目录
for f in memory/diary/*.md; do
  file_date=$(basename "$f" .md)
  if [[ "$file_date" < "$cutoff_date" ]]; then
    mv "$f" memory/diary/archived/
  fi
done
```

## 价值过滤标准

**保留**：
- 知识/经验/教训——"一个月后还需要知道这个"

**丢弃**：
- 过程描述（完成了什么）
- 技术实现细节（已写入代码）
- 可通过 `git log` 恢复的信息

## 自检清单

写入前自问：
1. **这是知识/经验/教训吗？** → 是则保留
2. **一个月后还需要知道吗？** → 是则保留
3. **能通过读代码推断吗？** → 是则丢弃
