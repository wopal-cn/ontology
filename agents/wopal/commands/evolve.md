# 自我进化

## 目标

从日记中提炼有价值的信息，沉淀到长期文件。

> 日记是暂存池，进化是真正的沉淀——不只是记录发生了什么，而是从经验中**学习**。

## 流程

### 1. 扫描信息源

同时扫描：
- **本次会话**：关键事件、决策、教训
- **短期记忆**：`memory/diary/` 下的日记

**跳过已标记条目**：`[进化吸收]` `[进化排除]`

### 2. 判断去向

| 信息类型 | 去向 | 判断标准 |
|---------|------|---------|
| 永久约束 | `AGENTS.md` | 空间级行为约束 |
| 项目规范 | 项目 `AGENTS.md` | 项目内部的技术栈、结构 |
| 工作空间知识 | `MEMORY.md` | 工作空间特有的事实 |
| 用户偏好 | `USER.md` | 用户的沟通/工作偏好 |

### 3. 展示方案

```
🧬 进化方案：

📝 MEMORY.md
  - [知识] OpenCode 非交互模式权限会被 auto-reject
  - [经验] skill 变更不再需要 sync-to-wopal.py 同步

📐 AGENTS.md
  - [规则] 项目修改后要在项目内执行 git 命令

📁 projects/agent-tools/AGENTS.md
  - [知识] 技能开发时 SKILL.md 需包含 description 字段

👤 USER.md
  - [偏好] "出方案" = 写方案文档，不是口头说明

确认执行？
```

### 4. 执行与标记

用户确认后：
1. 写入对应长期文件
2. 在日记条目开头加标签：
   - 已沉淀 → `[进化吸收]`
   - 无需沉淀 → `[进化排除]`

```markdown
- [进化吸收] [知识] OpenCode 不支持从其他目录加载 agents
- [进化排除] [经验] 渐进式加载架构（通用知识）
```

### 5. 日记归档

将已分析的日记文件移动到 `memory/diary/archived/`：

```bash
mv memory/diary/YYYY-MM-DD.md memory/diary/archived/
```

## 价值过滤标准

### 核心原则

**只有「换一个工作空间就不适用」的信息才值得沉淀。**

### 吸收 vs 排除

| 判断 | 吸收 ✓ | 排除 × |
|------|--------|--------|
| **知识** | 工作空间特有的目录结构、配置约定、工具行为 | 通用设计原则、架构模式、编程常识 |
| **经验** | 工作空间验证有效的具体做法 | 通用最佳实践 |
| **教训** | 工作空间特有的坑、反复踩过的雷 | 一次性失误、通用错误类型 |

### 常见排除项

| 类型 | 原因 |
|------|------|
| TODO/里程碑 | 任务状态，不是知识 |
| 已在代码 | 代码即记录 |
| 已有文档 | 文档即记录 |
| 通用知识 | 换工作空间也适用 |
| 一次性事件 | 无复用价值 |

### 判断自检

1. **换一个工作空间还需要知道吗？** → 不需要 → ✓ 吸收
2. **能从代码/文档直接推断吗？** → 能 → × 排除
3. **一个月后回头看还有用吗？** → 没有 → × 排除
