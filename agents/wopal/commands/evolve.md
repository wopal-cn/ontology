# 自我进化 (Evolve)

## 目标

从日记中提炼有价值的信息，沉淀到正确位置。

> 日记是暂存池，进化是真正的沉淀——不只是记录发生了什么，而是从经验中**学习**。

## 流程

### 1. 扫描信息源

同时扫描：
- **本次会话**：关键事件、决策、错误
- **短期记忆**：`memory/diary/` 下的日记

**跳过已标记条目**：`[进化吸收]` `[进化排除]`

### 2. 判断去向

参见 `AGENTS.md` 记忆与进化章节的"沉淀位置"和"优化方向"：

| 信息性质 | 沉淀位置 |
|----------|----------|
| 跨会话通用知识/经验 | `MEMORY.md` |
| 用户偏好 | `USER.md` |
| 工作规则/约定 | `AGENTS.md` |
| 项目特定知识 | 项目 `AGENTS.md` |
| 核心行为特质 | 灵魂（系统提示词） |
| 待优化问题 | `memory/PLAN.md`（用 plan-master 技能） |

### 3. 展示方案

```
进化方案：

📝 MEMORY.md
  + [知识] OpenCode API 在 labs/ref-repos/opencode/packages/sdk/openapi.json

👤 USER.md
  + [偏好] "出方案" = 写方案文档，不是口头说明

📐 AGENTS.md
  + [规则] 技能脚本必须先 cd 进目录再执行

📋 PLAN.md
  + [待优化] Fae 长任务 SSE 超时问题需研究

确认执行？
```

### 4. 执行与标记

用户确认后：
1. 写入对应长期文件
2. 在日记条目标记：
   - 已沉淀 → `[进化吸收: 简短原因]`
   - 无需沉淀 → `[进化排除: 简短原因]`

```markdown
- [进化吸收: 空间特有配置] OpenCode API 在 labs/ref-repos/opencode/packages/sdk/openapi.json
- [进化排除: 通用知识] 渐进式加载架构
```

### 5. 日记归档

将已分析的日记移动到 `memory/diary/archived/`

## 价值判断

参见 `AGENTS.md` 记忆与进化章节的"捕获三问"和"不记录"规则。

**核心原则**：
- 只沉淀与空间优化、项目建设**高度相关**且具有**长期复用价值**的信息
- 不记录通用知识、临时状态、会话流水、单纯的错误描述
