# 快速记录 (Memo)

## 内容: `$ARGUMENTS`

- **有参数**：直接使用参数内容
- **无参数**：扫描当前会话上下文，提取值得沉淀的信息

## 记录什么

日记是**暂存池**，只记录未来可能沉淀到长期文件的信息：

| 类型 | 说明 | 示例 |
|------|------|------|
| **[知识]** | 工作空间特有的工具/平台行为、配置约定 | "OpenCode 不支持从其他目录加载 agents" |
| **[经验]** | 工作空间验证有效的具体做法 | "skill 变更不再需要 sync-to-wopal.py 同步" |
| **[教训]** | 工作空间特有的坑、反复踩过的雷 | "项目修改后要在项目内执行 git 命令" |
| **[决策]** | 工作空间内的决策（尚未形成独立文档） | "日记目录调整为 memory/diary/" |
| **[偏好]** | 用户偏好信号 | ""出方案" = 写方案文档，不是口头说明" |

## 不记录什么

| 类型 | 原因 | 正确做法 |
|------|------|---------|
| TODO/任务 | 任务状态，不是知识 | 用 `todo-tracker` 技能 |
| 已在代码 | 代码即记录 | 不重复记录 |
| 已有文档 | 文档即记录 | 不重复记录 |
| 通用知识 | 换工作空间也适用 | 无需记住 |
| 一次性事件 | 无复用价值 | 不记录 |
| 里程碑 | 完成状态，不是知识 | 用任务追踪 |

## 操作步骤

1. **确定内容**（有参数用参数，无参数扫描会话）
2. **判断价值**（参考上方两表）
3. **写入日记**：
   ```bash
   cat >> memory/diary/YYYY-MM-DD.md <<'EOF'

   ### HH:MM 主题
   - [类型] 内容
   EOF
   ```
4. **确认**：`📝 已记录`
