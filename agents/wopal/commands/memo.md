# 快速记录 (Memo)

## 内容: `$ARGUMENTS`

- **有参数**：直接使用参数内容
- **无参数**：扫描当前会话上下文，提取值得沉淀的信息

## 操作步骤

1. **确定内容来源**（见上方）
2. **提取内容**：
   - **[知识]**：隐性知识、工具/平台的非显而易见行为 → 写入日记
   - **[经验]**：操作技巧、有效做法 → 写入日记
   - **[教训]**：踩坑记录、负向警示 → 写入日记
   - **[决策]**：临时决策、进度记录 → 写入日记
   - **[偏好]**：用户偏好信号 → 写入 `USER.md`
   - **TODO**：待办事项 → 使用 `todo-tracker` 技能进行记录和管理

## 偏好信号识别

扫描会话中的以下信号，提取用户偏好：

| 信号 | 示例 | 提取偏好 |
|------|------|---------|
| 对做法的反馈 | "你又搞错了，你有疑问时要确认" | 不确定时必须确认 |
| 沟通风格反馈 | "你现在写的我看不懂" | 不喜欢复杂实现 |
| 明确要求 | "记住这个"、"下次注意" | 直接记录 |
| 隐性表达 | 用户反复纠正同一类问题 | 总结规律性偏好 |
1. **写入日记**：
   ```bash
   cat >> memory/diary/YYYY-MM-DD.md <<'EOF'

   ### HH:MM 主题
   - [类型] 内容
   EOF
   ```
2. **确认**：`📝 已记录`

## 注意

- **不记录**：可从项目文档或代码中直接获取的信息、通用最佳实践
