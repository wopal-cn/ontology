# 记忆管理规则

## 核心原则

- **不主动记录**：只在用户明确指令（"记住这个"、"记下来"）或调用命令时才记录
- **价值导向**：只记录对子项目开发和工作空间演进有持续价值的信息
- **判断基准**："一个月后回头看，这条信息还有用吗？"

## 记录位置判断

| 优先级 | 文件 | 记录什么 | 判断问题 |
|--------|------|---------|---------|
| 1 | `SOUL.md` | Agent 身份与特质 | "这是关于**我是谁**的信息吗？" |
| 2 | `USER.md` | 用户偏好与习惯 | "这是关于**用户**的信息吗？" |
| 3 | `AGENTS.md` | 工作规则与红线 | "这是关于**我如何工作**的规则吗？" |
| 4 | `MEMORY.md` | **子项目架构决策、演进里程碑、教训、实施路线图** | "这对**子项目跨会话开发**有参考价值吗？" |
| 5 | `.workspace.md` | 目录结构变更事实 | "目录结构**发生了变更**吗？" |
| 6 | 子项目 `AGENTS.md` | 项目特有技术栈、模式 | "这是**特定项目内部**的技术细节吗？" |
| 7 | `memory/YYYY-MM-DD.md` | 默认——日常流水 | 以上都不符合时 |

### `.workspace.md` 与 `MEMORY.md` 的区别

- `.workspace.md`：记录**当前有什么**（目录结构事实），只陈述，不解释
- `MEMORY.md`：记录**子项目为什么这样演进**（决策背景、里程碑、教训），服务跨会话开发

## 记录流程

### 步骤 1：判断位置
根据上表判断目标文件。

### 步骤 2：提炼内容

**长期文件**（SOUL/USER/AGENTS/MEMORY/子项目 AGENTS）：
- 提炼为简洁陈述句，格式：`- [内容]`

**短期记忆**（memory/YYYY-MM-DD.md）：
```
### HH:MM 主题
- [类型] 要点
```

类型标记：`[决策]` `[里程碑]` `[教训]` `[TODO]` `[关键事务]`

### 步骤 3：确认并写入
展示记录方案（目标文件 + 内容），等待用户确认后写入。

## 示例

**"记住：我比较喜欢简洁的沟通方式"**
→ `USER.md` / 沟通偏好章节 / `- 偏好简洁的沟通方式`

**"记住：wopal 项目决定从 Pages Router 迁移到 App Router"**
→ `MEMORY.md` / 子项目开发进展 / `- wopal 项目决定从 Pages Router 迁移到 App Router，原因：...`

## 自我进化流程

**触发条件**：用户要求"自我进化"或"wopal-evolve"

### 步骤 1：扫描信息源

同时扫描两个来源：
- **本次会话**：回顾当前会话中的关键事件、决策、教训
- **短期记忆**：读取 `memory/` 下近 3 天的记忆流水

**跳过已标记条目**：
- `[进化吸收]` - 已被进化吸收到长期文件，跳过
- `[进化排除]` - 已被判断为不需要进化，跳过

### 步骤 2：提取进化素材

从信息源中识别以下类型的进化素材：

| 类型 | 说明 | 判断标准 |
|------|------|---------|
| **能力演进** | 新增的工作方式 | "现在**能做什么**之前不能做的事？" |
| **架构决策** | 结构性选择 | "为什么这样**组织**？" |
| **关键发现** | 需要记住的教训 | "踩过的坑，**如何避免**？" |
| **规则改进** | 永久约束 | "这是**约束**还是**赋能**？" |

### 步骤 3：分类到目标文件

按记录位置判断表确定目标文件（优先子项目规范，其次工作空间层面）。

### 步骤 4：展示进化方案

按文件分组展示提炼结果，等待用户确认。

### 步骤 5：执行写入与标记

用户确认后：
1. 将内容写入对应长期文件
2. 在日记源文件中标记已处理的条目：
   - 已进化的条目 → 在类型标记前加 `[进化吸收]`
   - 已判断无需进化的条目 → 在类型标记前加 `[进化排除]`
3. **核销已完成的 TODO**：
   - 已完成的 TODO → 改为 `[✅ 已完成]` 或删除
   - 保留未完成的 TODO
