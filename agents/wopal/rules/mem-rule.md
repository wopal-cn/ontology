# 记忆管理规则

## 核心原则

- **价值导向**：只记录无法从代码/文档直接发现的信息
- **判断基准**："一个月后回头看，这条信息还有用吗？"

## MEMORY.md 定位

记录**无法从代码/文档直接发现**的信息：

| 类型 | 说明 | 例子 |
|------|------|------|
| **隐性知识** | 工具/平台的非显而易见行为 | OpenCode 非交互模式权限被 auto-reject |
| **操作经验** | 实践中摸索的技巧 | 委派 fae 时 description 简洁、prompt 详细 |
| **负向警示** | 踩坑记录，避免重蹈覆辙 | fae 长时间无新消息可能卡住 |

**不记录**：
- 架构决策（读代码/目录结构可知）
- 通用最佳实践（系统提示词已覆盖）
- 可从文档直接获取的信息

## 记录位置

| 优先级 | 文件 | 记录什么 |
|--------|------|---------|
| 1 | `USER.md` | 用户偏好、沟通方式 |
| 2 | `AGENTS.md` | 行为规则、安全红线 |
| 3 | `MEMORY.md` | 隐性知识、操作经验、负向警示 |
| 4 | 子项目 `AGENTS.md` | 项目特有技术细节 |
| 5 | `memory/diary/YYYY-MM-DD.md` | 日常流水（默认） |

## 主动进化机制

### 触发场景

| 场景 | 行为 |
|------|------|
| 会话中发现高价值信息 | 立即提示："发现高价值信息，是否沉淀？" |
| 会话结束时 | 扫描本次会话，如有未沉淀信息，主动提醒 |
| 用户说"记住"类指令 | 直接执行沉淀流程 |

### 高价值信息判定

满足任一条件即视为高价值：
1. 用户明确标记："记住这个"、"这个很重要"
2. 隐式发现：调试/探索中发现的非显而易见行为
3. 踩坑记录：报错、失败、返工后得出的教训
4. 操作技巧：实践中摸索出的高效做法

### 主动提醒话术

```
📝 发现高价值信息：
> [信息摘要]

建议沉淀到：[目标文件]
是否记录？[Y/n]
```

## 命令

| 命令 | 用途 |
|------|------|
| `/memo <内容>` | 快速记录到短期记忆 |
| `/evolve` | 从短期记忆提取高价值信息到长期文件 |

## 进化流程（/evolve）

### 步骤 1：扫描信息源

同时扫描：
- **本次会话**：关键事件、决策、教训
- **短期记忆**：`memory/` 下近 3 天的记忆流水

跳过已标记条目：`[进化吸收]` `[进化排除]`

### 步骤 2：提取进化素材

识别以下类型：

| 类型 | 判断标准 |
|------|---------|
| 隐性知识 | "这是**非显而易见**的行为吗？" |
| 操作经验 | "这是**实践中摸索**的技巧吗？" |
| 负向警示 | "这是**踩过的坑**吗？" |

### 步骤 3：分类到目标文件

按记录位置表确定目标文件。

### 步骤 4：展示进化方案

按文件分组展示，等待用户确认。

### 步骤 5：执行写入与标记

用户确认后：
1. 写入对应长期文件
2. 在源文件中标记：`[进化吸收]` 或 `[进化排除]`
3. 核销已完成的 TODO

## 快速记录流程（/memo）

1. 确定目标文件：`memory/diary/YYYY-MM-DD.md`
2. 获取当前时间
3. 追加内容（无需确认）
4. 确认完成
