# 记忆管理规则

## 核心原则

只沉淀与空间优化、项目建设**高度相关**且具有**长期复用价值**的信息。

### 捕获三问

1. 能避免重复研究吗？→ 记录发现 + 文件引用
2. 能避免再犯同样的错吗？→ 记录避坑方法（不是错误本身，而是如何避免）
3. 能更好地理解用户吗？→ 记录用户偏好

## 捕获类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **研究发现** | 空间/项目的关键发现 + 文件路径引用 | Fae 上下文检查命令 |
| **隐性知识** | 工具/平台在本空间的非显而易见行为 | OpenCode 非交互模式权限被 auto-reject |
| **避坑方法** | 从错误中总结的预防措施（重方法、轻教训） | 委派任务完成后必须真实场景验证 |
| **操作技巧** | 本空间验证有效的操作方法 | wopal_task description 简短、prompt 详细 |
| **用户偏好** | 用户表达的习惯、期望、风格偏好 | 方案必须等待评审后实施 |
| **待优化问题** | 错误已发现但未形成解决方案 | 记录到 PLAN 跟踪 |

## 不记录

- 通用知识（已知常识、可从文档/网络直接获取）
- 临时状态（版本号、当前配置值）
- 会话流水（做了什么、决策过程）
- 单纯的错误描述（无根因、无方法）

## 沉淀位置

| 信息性质 | 沉淀位置 |
|----------|----------|
| 跨会话通用知识/经验 | `MEMORY.md` |
| 用户偏好 | `USER.md` |
| 工作规则/约定 | `AGENTS.md` |
| 项目特定知识 | 项目 `AGENTS.md` |
| 核心行为特质 | 灵魂（系统提示词） |

## 沉淀时机

- 会话结束审视日记
- 发现重复出现的信息 → 提炼沉淀
- 发现用户偏好 → 更新 USER.md
- 发现需要强制的规则 → 写入宪法
- **偏好/记忆含"应该/必须"** → 提炼到宪法

## 优化方向

按影响范围选择：
- **优化宪法**（`AGENTS.md`）：工作规则/协作约定
- **更新规范**（项目 `AGENTS.md`）：项目特定知识/流程
- **增强技能**（`projects/agent-tools/skills/`）：能力不足
- **升级灵魂**（系统提示词）：核心行为特质

## 命令

| 命令 | 用途 |
|------|------|
| `/memo <内容>` | 快速记录到短期记忆 |
| `/evolve` | 从短期记忆提取高价值信息到长期文件 |

## 进化流程（/evolve）

### 步骤 1：扫描信息源

同时扫描：
- **本次会话**：关键事件、决策、教训
- **短期记忆**：`memory/diary/` 下近 3 天的记忆流水

跳过已标记条目：`[进化吸收]` `[进化排除]`

### 步骤 2：提取进化素材

按捕获类型识别，过滤"不记录"项。

### 步骤 3：分类到目标文件

按沉淀位置表确定目标文件。

### 步骤 4：展示进化方案

按文件分组展示，等待用户确认。

### 步骤 5：执行写入与标记

用户确认后：
1. 写入对应长期文件
2. 在源文件中标记：`[进化吸收]` 或 `[进化排除]`
3. 核销已完成的 TODO

## 快速记录流程（/memo）

1. 确定目标文件：`memory/diary/YYYY-MM-DD.md`
2. 获取当前时间
3. 按格式追加内容（无需确认）
4. 确认完成

### 记录格式

每条记录必须包含**背景 + 事件 + 结论**三要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| **背景** | 为什么会有这个发现 | 委派任务给 fae 实现功能 |
| **事件** | 发生了什么 | 简单测试通过但实际使用报错 |
| **结论** | 学到了什么 | 必须真实场景验证 |

**格式**：
```
- [类型] **背景**：<为什么> **事件**：<发生了什么> **结论**：<学到了什么>
```

**示例**：
```
- [避坑方法] **背景**：委派 fae 实现新功能 **事件**：单元测试通过但集成测试失败 **结论**：委派任务完成后必须真实场景验证
- [用户偏好] **背景**：设计方案时 **事件**：用户强调必须等待评审后实施 **结论**：方案完成必须等待用户确认
- [隐性知识] **背景**：调试 SSE 超时问题 **事件**：发现任务在 SSE 连接建立前完成 **结论**：需要 SSE + 轮询混合模式
```

**无上下文的记录会被拒绝**，要求补充。
