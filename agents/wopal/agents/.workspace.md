# .workspace.md — wopal-workspace 工作空间导航地图

> 本文件为 Agent 提供工作空间快速全景认知。详细行为规则参阅 `AGENTS.md`。

---

## 一、核心项目区 (`projects/`)

各子项目拥有**独立的 Git 历史与生命周期**，不得跨项目污染提交历史。

| 路径 | 名称 | 定位 | 状态 |
|------|------|------|------|
| `projects/agent-tools/` | AI 工具研发中心 | 命令 / 规则 / 技能 / 插件的**源码仓库** | 活跃 |
| `projects/web/wopal/` | Wopal 平台 | Astro 前端（博客 + 教程 + 产品） | 持续迭代 |
| `projects/python/flex-scheduler/` | 任务调度系统 | 后端服务，规划重构为 `wopal-admin` | 规划中 |

> ⚠️ 修改命令 / 规则 / 技能时，**必须**在 `projects/agent-tools/` 源码中操作，然后部署到 `.wopal/`。

---

## 二、Agents 工具部署区 (`.wopal/`)

`.wopal/` 是所有 Agent 平台的**共享运行时部署站**，内容由 `projects/agent-tools/` 部署而来。

| 子目录 | 用途 | 来源 |
|--------|------|------|
| `commands/` | 已部署自定义命令（斜杠命令） | copy 自 `agent-tools/commands` |
| `rules/` | 已部署条件触发规则 | copy 自 `agent-tools/rules` |
| `skills/` | 已部署技能 | copy 自 `agent-tools/my-skills + download` |
| `agents/` | 已部署 agent 定义文件 | copy 自 `agent-tools/subagents` |

---

## 三、Agent 适配

各 agent 通过软链接读取 `.wopal/` 的共享资源：

| 配置路径 | agent | 说明 |
|----------|------|------|
| `.agents/` | 通用(多数 agent 已支持) | `commands` / `rules` / `skills` 链接自 `.wopal/` |
| `.claude/` | Claude Code | `commands` / `rules` / `skills` 链接自 `.wopal/` |
| `.opencode/` | OpenCode | `commands` / `rules` 链接自 `.wopal/` |
| `.kilocode/` | KiloCode | `workflows` / `rules` / `skills` 链接自 `.wopal/` |
| `.github/` | GithubCopilot | `skills` 链接自 `.wopal/` |

**Agent 适配层会根据不同 agent 的功能变化进行调整**

## 四、工作空间管理区

### Worktree 开发区
| 路径 | 用途 |
|------|------|
| `.worktrees/` | 工作空间级 worktree 统一管理区（使用 `.agents/skills/git-worktrees/scripts/worktree.sh` 管理） |

**路径格式**：`.worktrees/<project>-<branch>`（如 `agent-tools-feature-auth`）
**管理方式**：工作空间级统一管理，支持动态项目验证和 OpenSpec 工作流集成
**详细指南**：参见 `.agents/skills/git-worktrees/SKILL.md`

### Wopal 记忆系统
| 文件/目录 | 角色 |
|-----------|------|
| `SOUL.md` | Wopal 的身份与特质定义 |
| `USER.md` | 用户偏好与沟通方式 |
| `AGENTS.md` | 工作空间行为宪法（强制行为规则，AI Agent 专用） |
| `MEMORY.md` | 长期记忆 （空间和子项目重大事件和事实记忆库）|
| `memory/*.md` | 短期记忆（每日流水） |
