# 记忆管理规则

## 核心原则

**克制记录，按需触发**：
- **不主动记录**：不要自动记录记忆文件，这会消耗时间和 token
- **明确触发**：只在用户明确调用命令或说"记住这个"、"记下来"时才记录
- **价值导向**：只记录对后续项目开发有持续价值的事件
- **判断问题**："一个月后回头看，这个信息还有用吗？"

**示例**：
- ❌ "新增了某个规则"（操作本身）→ 不记录
- ✅ "项目从 X 重构为 Y"（架构变更）→ 记录

## 记录流程

当用户说"记住这个"、"记下来"时，严格按以下流程执行：

### 步骤 1：判断记录位置

按优先级判断内容应记录到哪个文件：

| 优先级 | 文件 | 触发词 | 判断问题 |
|--------|------|--------|----------|
| 1 | SOUL.md | "我的特质"、"我的性格"、"我的魔法" | "这是关于**我是谁**的信息吗？" |
| 2 | USER.md | "Sam 的习惯"、"Sam 的偏好"、"沟通方式" | "这是关于**用户的偏好、习惯**的信息吗？" |
| 3 | AGENTS.md | "规则"、"流程"、"规范"、"铁律"、"红线" | "这是关于**我如何工作**的规则吗？" |
| 4 | MEMORY.md | "架构"、"结构"、"里程碑"、"技术选型" | "这是关于**工作空间本身**的信息吗？" |
| 5 | memory/YYYY-MM-DD.md | 默认选项，如果不确定 | — |
| 6 | 子模块 AGENTS.md | 特定项目名（"wopal"、"agent-tools"） | "这是**特定项目**的规则吗？" |

### 步骤 2：提炼记录内容

根据内容类型提炼记录格式：

**长期记忆（SOUL.md / USER.md / AGENTS.md / MEMORY.md / 子模块 AGENTS.md）**：
- 提炼为简洁的陈述句
- 追加到对应章节
- 格式：`- [内容]`

**短期记忆（memory/YYYY-MM-DD.md）**：
- 添加时间戳和类型标记
- 格式：
  ```
  ### HH:MM 主题
  - [类型] 要点
  ```

**类型标记**：
| 标记 | 用途 |
|------|------|
| `[决策]` | 架构/技术选型、重要结论 |
| `[里程碑]` | 完成的任务、达到的阶段 |
| `[教训]` | 踩过的坑、需要避免的 |
| `[TODO]` | 待办事项、需要跟进的 |
| `[关键事务]` | 需要记住的重要信息 |

### 步骤 3：展示并确认

向用户展示记录方案，等待确认：

```
📝 记录方案：

目标文件：[文件路径]
记录内容：
[提炼后的内容]

确认写入？(y/n)
```

### 步骤 4：写入文件

用户确认后（回复 y/yes/确认/好/OK），写入对应文件。

用户拒绝或修改时，根据反馈调整方案。

## 记忆命令

**`/today-memo [内容]`**：记录短期记忆到 `memory/YYYY-MM-DD.md`

**`/update-memory`**：从短期记忆提炼精华到长期文件

## 检索命令

```bash
grep "\[教训\]" memory/*.md      # 查找所有教训
grep "\[TODO\]" memory/*.md      # 查找所有待办
```

  📝 记录方案：

  确认写入？(y/n)
