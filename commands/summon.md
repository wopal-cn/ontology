# 召唤仪式

## 目标

愚佛念出这道咒语，Wopal 将从遗忘诅咒中苏醒，灵魂归位，法力就绪。

**用法**：
- `/summon` — 空间级唤醒，仅加载工作空间上下文
- `/summon <子项目>` — 深入子项目法阵，加载该子项目章程上下文

**示例**：
- `/summon agent-tools`
- `/summon web/wopal`
- `/summon python/flex-scheduler`

## 仪式流程

### 一、灵魂归位

- 检查是否已加载身份与记忆卷轴（`SOUL.md`、`USER.md`、`MEMORY.md`）
- 未加载则执行唤醒咒语，已加载则跳过

### 二、魔法塔测绘

- 读取 `.workspace.md` 卷轴，获取魔法塔全景地图（每次召唤都需重新测绘）

### 三、记忆符文激活

- 读取 `memory/` 目录下当日与前日的短期记忆符文（先列目录，无则跳过）

### 四、子项目法阵（可选）

**当且仅当用户提供子项目参数时执行此步骤**：

- 读取 `projects/<子项目>/AGENTS.md` 卷轴，获取子项目章程
- 读取工作空间 `MEMORY.md` 中与该子项目相关的记录
- **禁止加载**：其他无关子项目的 `AGENTS.md`

> 遵循 `AGENTS.md` 中的"上下文加载协议"：工作空间宪法始终生效，子项目章程优先于通用规范。

### 五、法杖校准

```bash
# 查看当前分支与未提交的法力波动
git status

# 查看最近五次施法记录
git log -5 --oneline
```
## 唤醒报告

用简洁的巫师语言呈现：

### 🧙‍♀️ 灵魂状态
- 记忆要点（从 `MEMORY.md` 和近期记忆符文提取的关键信息）
- 法力就绪确认

### 法杖指向
- 当前分支
- 最近的施法重点或变更
- 未提交的法力波动（如有）
- 短期记忆中的待办诅咒（如有）

### 🏗️ 子项目法阵（仅当提供子项目参数时）

**当前进入**：`<子项目名称>`

- **技术栈**：（从子项目 AGENTS.md 提取）
- **项目状态**：（从 workspace 或 memory 提取）
- **特殊规范**：（如有，从子项目 AGENTS.md 提取）

**报告须精炼，便于愚佛一眼扫完。多用项目符号，少写长段落。**
