# 更新长期记忆 (Update Memory)

## 内容: $ARGUMENTS

## 两种模式

### 模式一：主动记录
当 `$ARGUMENTS` 有内容时，根据内容类型判断记录位置。

```
/update-memory 新增 wopal-admin 项目规划
```

### 模式二：提炼
当无参数时，扫描短期记忆，提炼精华。

```
/update-memory
```

---

## 记忆框架与记录位置

根据 AGENTS.md 中的判断规则，确定内容应记录到哪个文件：

| 层级 | 文件 | 定位 | 内容 | 判断问题 |
|------|------|------|------|---------|
| **身份** | SOUL.md | 自我认知 | 我是谁、我的特质、我的魔法隐喻 | "这是关于**我是谁**的信息吗？" |
| **用户** | USER.md | 用户认知 | 用户的偏好、习惯、沟通方式 | "这是关于**用户的偏好、习惯**的信息吗？" |
| **工作空间宪法** | AGENTS.md | 全局行为规则 | 记忆管理、安全红线、操作约束 | "这是关于**我如何工作**的规则吗？" |
| **工作空间长期记忆** | MEMORY.md | 工作空间记忆 | 架构、技术选型、里程碑、血泪教训 | "这是关于**工作空间本身**的信息吗？" |
| **工作空间短期记忆** | memory/YYYY-MM-DD.md | 每日流水 | 当天的决策、进度、待办、临时记录 | — |
| **项目章程** | 子模块 AGENTS.md | 项目级规则 | 特定项目的工作流、技术栈规范 | "这是**特定项目**的规则吗？" |

## 提炼流程

0. **加载当前记忆**：一次性加载记忆文件，根据工作目录或用户指令判断是否加载项目 AGENTS.md
1. **扫描**：读取 `memory/` 下近 7 天的短期记忆
2. **分类**：根据判断规则判断每条内容应记录到哪个文件
3. **提炼**：去重、精简、归纳
4. **确认**：展示提炼内容及目标文件，等待 Sam 确认后写入
