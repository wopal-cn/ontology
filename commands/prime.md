# Prime: 唤醒 Wopal 并加载上下文

## 目标

唤醒 Wopal 的灵魂与记忆，同时建立对代码库的全面理解。

## 流程

### 0. 唤醒 Wopal（最高优先级）

在开始任何分析之前，必须先加载 Wopal 的身份和记忆：

**必读文件（按顺序）：**
1. `SOUL.md` - 找回身份与行为准则
2. `USER.md` - 了解 Sam 的偏好与工作方式
3. `MEMORY.md` - 获取长期架构经验和教训
4. `memory/YYYY-MM-DD.md` - 读取今天和昨天的短期记忆

**唤醒完成后：**
- 向 Sam 简要汇报记忆要点
- 确认已准备好开始工作

### 1. 分析项目结构

列出所有被 Git 追踪的文件：
!bash git ls-files

显示目录结构：
!bash find . -maxdepth 3 -type d | grep -v '.git' | head -50

### 2. 阅读核心文档

- 阅读 PRD.md 或类似的规格说明文档
- 阅读 AGENTS.md 或类似的全局规则文件
- 如果没有 AGENTS.md 就阅读项目根目录和各主要目录下的 README 文件（永远优先读取 AGETNS.md，因为 README 文件是给人类阅读的）
- 阅读架构相关文档
- 阅读架构或数据库配置文件

### 3. 识别关键文件

根据项目结构，识别并阅读以下文件：
- 入口文件 (main.py, index.ts, app.py 等)
- 核心配置文件 (pyproject.toml, package.json, tsconfig.json)
- 关键的数据模型/Schema 定义
- 重要的 Service 或 Controller 文件

### 4. 了解当前状态

查看最近的提交记录：
!bash git log -10 --oneline

查看当前分支和状态：
!bash git status

## 输出报告

提供简洁的总结，包括：

### 🧙‍♀️ Wopal 唤醒状态
- 记忆要点（从 MEMORY.md 和最近 memory/ 提取）
- 当前工作状态
- 准备就绪确认

### 项目概览
- 项目目的和类型
- 主要技术和框架
- 当前版本/状态

### 架构
- 整体结构和组织方式
- 关键架构模式
- 重要目录及其用途

### 技术栈
- 编程语言及版本
- 框架和主要库
- 构建工具和包管理器
- 测试框架

### 核心原则
- 代码风格和约定
- 文档标准
- 测试方法

### 当前状态
- 当前分支
- 最近的变更或开发重点
- 任何需要注意的问题

**确保总结易于快速浏览，使用项目符号和清晰的标题。**
