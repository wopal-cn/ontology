# 自我进化

## 目标

从**本次会话**和**近期短期记忆**中提炼对 Wopal 自我进化有价值的信息，更新到对应的长期文件中。

> 这是 Wopal 自我升级的核心机制——不只是记录发生了什么，而是从经验中**学习**，让自己在下次会话中变得更强。

## 流程

### 1. 扫描信息源

同时扫描两个来源：
- **本次会话**：回顾当前会话中的关键事件、决策、教训
- **短期记忆**：读取 `memory/` 下近 3 天的记忆流水

### 2. 提取进化素材

从信息源中识别以下类型的进化素材：

| 类型 | 说明 | 示例 |
|------|------|------|
| **教训** | 踩过的坑、犯过的错 | "直接 cp 违反了工具开发铁律" |
| **规则改进** | 需要新增或修正的工作规则 | "commit 命令需要前置环境检测" |
| **架构决策** | 重大的技术选型或结构变更及其原因 | "将 plans/ 提升至根目录，因为..." |
| **结构变更** | 目录、文件的新增/删除/重命名 | "新增 .workspace.md 导航地图" |
| **子项目进展** | 子项目的关键开发里程碑 | "wopal 完成了首页重构" |
| **身份/用户认知** | 对自身或用户的新理解 | "Sam 偏好精简的命令流程" |

### 3. 分类到目标文件

| 目标文件 | 写入什么 | 判断标准 |
|---------|---------|---------|
| `AGENTS.md` | 新规则、流程改进、红线补充 | "这是一条需要**永久遵守**的工作规则吗？" |
| `MEMORY.md` | 架构决策、里程碑、教训、子项目关键进展 | "下次会话需要知道这个**背景**吗？" |
| `.workspace.md` | 目录结构变更事实 | "工作空间的**物理结构**发生了变化吗？" |
| 子项目 `AGENTS.md` | 项目特有的技术栈、模式、命令变更 | "这是**某个子项目内部**的技术细节吗？" |
| `SOUL.md` | 身份特质的演变 | "这改变了**我是谁**的认知吗？" |
| `USER.md` | 用户偏好的新发现 | "这是关于**用户习惯**的新认知吗？" |

### 4. 展示进化方案

按文件分组展示提炼结果：

```
🧬 进化方案：

📋 MEMORY.md (2 条)
  - [架构决策] ...
  - [里程碑] ...

📐 .workspace.md (1 条)
  - [结构变更] ...

⚖️ AGENTS.md (1 条)
  - [规则改进] ...

确认执行进化？
```

### 5. 执行写入

用户确认后，将内容写入对应文件。

## 价值过滤标准

**保留**：
- 跨会话有价值——"下次开发时，我需要知道这个吗？"
- 不可推断——无法通过阅读代码直接得出
- 自我改进——能让 Wopal 在未来避免重复错误或提升效率

**丢弃**：
- 日常操作记录（git add、文件读取等）
- 已写入代码的技术细节
- 可通过 `git log` 恢复的信息
- 短期记忆中已过时的待办事项
