# 自我进化

## 目标

从**本次会话**和**近期短期记忆**中提炼对 Wopal 自我进化有价值的信息，更新到对应的长期文件中。

> 这是 Wopal 自我升级的核心机制——不只是记录发生了什么，而是从经验中**学习**，让自己在下次会话中变得更强。

## 核心原则

**记录结果与影响，而非过程与技术细节。**

| 应该记录 | 不应记录 |
|---------|---------|
| 新增了什么能力 | 完成了几个阶段 |
| 对工作流程带来什么变化 | 用了什么技术栈 |
| 为什么做这个决策 | 如何实现的技术细节 |
| 避免了什么问题 | 做了哪些操作步骤 |

**子项目变更归子项目章程。**

- 子项目内部的结构、技术栈、工具变更，应更新子项目的 `AGENTS.md`
- 工作空间 `.workspace.md` 只记录顶层项目清单，不记录子项目内部细节
- 工作空间 `MEMORY.md` 只记录跨子项目的架构决策和能力变化

## 流程

### 1. 扫描信息源

同时扫描两个来源：
- **本次会话**：回顾当前会话中的关键事件、决策、教训
- **短期记忆**：读取 `memory/` 下近 3 天的记忆流水

**跳过已标记条目**：
- `[进化吸收]` - 已被进化吸收到长期文件，跳过
- `[进化排除]` - 已被判断为不需要进化，跳过

### 2. 提取进化素材

从信息源中识别以下类型的进化素材：

| 类型 | 说明 | 判断标准 | 正确示例 | 错误示例 |
|------|------|---------|---------|---------|
| **能力演进** | 新增的工作方式 | "现在**能做什么**之前不能做的事？" | "可驱动 Agent 后台执行长任务" | "完成了 5 个开发阶段" |
| **架构决策** | 结构性选择 | "为什么这样**组织**？" | "三层解耦架构：源码→部署站→适配" | "使用 Node.js 技术栈" |
| **关键发现** | 需要记住的教训 | "踩过的坑，**如何避免**？" | "OpenCode 非交互模式权限会被 auto-reject" | "appendOutput 有时序问题（已修复）" |
| **规则改进** | 永久约束 | "这是**约束**还是**赋能**？" | "commit 前必须检查子项目状态" | "优化了 commit 流程" |

**区分规则 vs 能力：**
- 规则（约束）→ 归 `AGENTS.md`，如"目录保护"、"禁止修改"
- 能力（赋能）→ 归 `MEMORY.md`，如"可后台执行"、"可协作编排"

### 3. 分类到目标文件

| 目标文件 | 写入什么 | 判断标准 |
|---------|---------|---------|
| `AGENTS.md` | 永久遵守的工作规则 | "这是一条需要**永久遵守**的规则吗？" |
| `MEMORY.md` | **跨子项目**的能力变化、架构决策、教训 | "这改变了我**能做什么**或**如何工作**吗？且**影响跨子项目**？" |
| `.workspace.md` | 顶层项目清单变更 | "工作空间的项目列表**发生变化**吗？" |
| 子项目 `AGENTS.md` | 子项目内部的技术栈、目录结构、工具状态 | "这是**某个子项目内部**的变更吗？" |
| `SOUL.md` | 身份特质的演变 | "这改变了**我是谁**的认知吗？" |
| `USER.md` | 用户偏好的新发现 | "这是关于**用户习惯**的新认知吗？" |

**优先级**：子项目内部的变更优先更新子项目章程，避免在工作空间层面零散记录。

### 4. 展示进化方案

按文件分组展示提炼结果：

```
🧬 进化方案：

📋 MEMORY.md (2 条)
  - [能力变化] 新增后台进程管理能力，现可驱动 Agent 在后台执行长任务
  - [架构决策] 将 process 独立成包，因复用性和可维护性需求

📐 .workspace.md (1 条)
  - [结构变更] 新增 tools/process/ 目录

⚖️ AGENTS.md (1 条)
  - [规则改进] 技能调用必须从 .agents/skills/ 加载

确认执行进化？
```

### 5. 执行写入与标记

用户确认后：
1. 将内容写入对应长期文件
2. 在日记源文件中标记已处理的条目：
   - 已进化的条目 → 在类型标记前加 `[进化吸收]`
   - 已判断无需进化的条目 → 在类型标记前加 `[进化排除]`

**标记示例**：
```markdown
### 14:04 @wopal/process 工具开发完成
- [进化吸收] [里程碑] 完成所有 5 个开发阶段
- [进化吸收] [决策] 使用 node-pty 作为可选依赖
- [进化排除] [教训] appendOutput 异步写入有时序问题（已修复，无需长期记忆）
- [TODO] 修复剩余 5 个测试用例  ← 未标记（待办暂不处理）
```

## 价值过滤标准

**保留**：
- 能力变化与影响——"这改变了我能做什么或如何工作"
- 永久规则——"下次会话需要遵守这个吗？"
- 不可推断的教训——"无法通过代码直接得出的经验"

**丢弃**：
- 过程描述（完成了什么、做了什么）
- 技术实现细节（已写入代码）
- 可通过 `git log` 恢复的信息
- 已解决的问题（除非需要记住教训）
- 短期记忆中已过时的待办事项

## 自检清单

写入前自问：
1. **这是结果还是过程？** → 只保留结果
2. **这能改变我的能力或工作方式吗？** → 是则保留
3. **一个月后还需要知道这个吗？** → 是则保留
4. **这能通过读代码推断吗？** → 是则丢弃
